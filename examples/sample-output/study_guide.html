    Overview

    sample-cli is a small Python command-line tool that fetches JSON from public APIs, normalizes it, and stores it in a local SQLite database for later reporting.

    Purpose

    Built to give analysts a friction-free way to snapshot third-party API data without standing up an ETL pipeline. Replaces ad-hoc curl | jq | sqlite3 shell scripts with a typed, testable Python flow.

    Architecture

    Step-by-Step Guide

    1. User runs sample-cli fetch --source github.
    2. cli.py parses args with argparse and loads config via config.py.
    3. The fetch command calls fetcher.get(source) which issues an HTTP request and returns raw JSON.
    4. parser.normalize(payload) flattens nested fields and coerces types.
    5. storage.insert(rows) writes to ~/.sample-cli/data.db using sqlite3.
    6. Running sample-cli report later calls reporter.render() which queries the DB and prints a formatted table.

    Core Modules

    FileRole
    cli.pyEntry point. Argparse setup and command dispatch.
    config.pyLoads ~/.sample-cli/config.toml with sensible defaults.
    fetcher.pyThin httpx wrapper with retry + timeout.
    parser.pyNormalization rules per source. Pure functions.
    storage.pySQLite schema + insert/query helpers.
    reporter.pyRenders query results as a rich table.

    Your first tasks

    1. Run sample-cli fetch --source github --repo torvalds/linux; done when the command stores one row in ~/.sample-cli/data.db.
    2. Add a parser test for a nested API payload; done when the test proves parser.normalize(payload) emits flat rows.
    3. Run sample-cli report --last 1; done when stdout shows the newest stored record.
