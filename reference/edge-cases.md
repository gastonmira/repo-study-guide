# Edge Cases

Support reference for `SKILL.md`. Read when the target repo does not fit the
happy path.

## Monorepos

Detect `workspaces` (`package.json`), `pnpm-workspace.yaml`, Turborepo, Nx,
Cargo workspaces. When the graph is available, `list_communities_tool` often
surfaces logical packages as distinct communities — cross-reference with
workspace config.

Generate one study guide *per workspace package* under
`docs/<package>/study_guide.html`, plus a root index.

## No README

Derive purpose from package metadata, top-level comments, and folder names.
Mark the "Purpose" section as `(inferred)`.

## Empty / scaffolding only

Produce a minimal guide noting the repo is a scaffold; suggest next steps
instead of forcing analysis.

## Unknown language

Fall back to file-extension stats and any build configs found.

## Update mode

- **Old guide without meta tags** (generated by a pre-update-mode template):
  fall back to parsing the hash from the visible `commit <code>...</code>`
  header text. If that is also missing, do a full regeneration.
- **Invalid hash / rebased history / shallow clone**: `git cat-file -e` fails →
  full regeneration, mentioning the reason in the final response.
- **Output dir changed between runs**: if the user asks for a different output
  directory than the existing guide's, treat it as an initial generation —
  do not diff against a guide in another directory.
- **Monorepos**: filter the `git diff --name-status` output per workspace
  package; only regenerate the guides of packages that were touched. The root
  index is refreshed whenever any package guide changes.
